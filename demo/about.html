HTML-Feedback

A Claude Code skill that turns a folder of static HTML pages into a collaborative review surface.

What it does

Point the skill at a directory of HTML files. It serves them locally, injects a small commenting layer into each page, and watches a JSONL inbox for feedback. You highlight text, click elements, or leave page-level notes; Claude reads the inbox, edits the source HTML directly, and the browser auto-reloads with the changed regions highlighted. Every edit is snapshotted and one-click revertible.

Core features

Text comments

Highlight any phrase and attach feedback to that exact range.

Element comments

Hover-and-click any element — image, table, section — to comment on it.

Page notes

For feedback that isn't tied to one spot.

Threaded replies

Push back on Claude's edit; the conversation stays on the same anchor.

Auto-reload

Pages refresh the moment a new edit lands and walk you through what changed.

Per-edit revert

Every edit takes a snapshot. One click restores it.

Orphan re-pinning

If an anchor can't be found after an edit, the comment lands in the orphan tray for you to re-pin.

Local-only, token-gated

Binds 127.0.0.1, every request carries a random per-session token.

How to run it

python scripts/start.py --dir <folder-with-html>

Or, inside Claude Code, just say "start html feedback on ./pages". The server prints a URL with a token; your default browser opens to it.

What lives where

  1. lib/server.py — Stdlib HTTP server with SSE, token auth, snapshot/revert, idle & parent-PID shutdown.
  2. lib/feedback.js — Client library: selection toolbar, sidebar, anchor resolution, walkthrough.
  3. lib/feedback.css — Themed UI, respects prefers-color-scheme.
  4. scripts/inject.py — Idempotent injector that adds the lib tags to each HTML file.
  5. scripts/start.py — One-command launcher: inject, spawn server, open browser, handle Ctrl-C.
  6. SKILL.md — Tells Claude Code when and how to invoke the skill.
  7. SPEC.md — The wire-protocol & schema contract.
Security note. The server binds 127.0.0.1 only and requires a per-session token on every request. It self-shuts down when idle or when its parent process dies. It is intended for single-user local use; do not expose the port to a network.

Try it on this page

This page is a working demo — everything described above is something you can do right now without leaving the tab. Open the sidebar (💬 button in the corner), highlight a sentence or click an element, then leave a comment.

A few things worth trying:

  • Text comments — highlight any phrase, even across paragraphs, and request a change. The anchor is robust to surrounding edits.
  • Element comments — press e to enter element-pick mode, then click an image, table, or code block. Your comment is attached to that element, not a text range.
  • Page-level notes — the + Page button in the sidebar header is for feedback that isn't tied to a specific spot.
  • Batch processing — comments accumulate quietly. When you're ready, hit the blue ▶ Process (N) button to send the whole batch to Claude in one shot.
  • Threaded replies — after Claude addresses a comment, the sidebar shows it as ✓ addressed. Hit Reply to push back or refine; the conversation stays on the same anchor.
  • Walkthrough — after the page auto-reloads, yellow highlights mark the changed regions. Press j / k to step through them.
  • Revert & Redo — each addressed comment in the sidebar has a small ↶ Revert link. After reverting, the link flips to ↺ Redo so you can re-apply the change in one click. Both use patch-style diffs, so reverting one edit doesn't disturb the others.
  • Orphans — if an anchor becomes unfindable after structural edits, the comment lands in the orphan tray with a Re-pin button so you can point it at a new location.

When you're done, hit Ctrl+C in the terminal running the server. The server also self-shuts on idle, so closing the tab is just as fine.

Built as an internal tool. Python 3.10+ · no external dependencies · works on Windows, macOS, Linux.
